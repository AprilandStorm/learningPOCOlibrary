# foundation
## logger
- 特殊的通道（channel)，日志框架的只要入口点
- 使用Logger类的实例来生成日志消息，并发送到最终目的地。
- 实例以树状结构组织，除根Logger外，每个Logger对象只有一个直接祖先。新创建的Logger都从其直接祖先继承其属性（通道和级别）
- 每个logger都连接一个通道
- 每个logger都有一个日志级别，根据优先级过滤消息。只有优先级等于或高于指定级别的消息才会被传递。例如，如果 Logger 的级别设置为 3 (PRIO_ERROR)，则只有优先级为 PRIO_ERROR、PRIO_CRITICAL 和 PRIO_FATAL 的消息才会被传播。如果级别设置为 0，则该 Logger 将被禁用。
- logger的名称决定了它在记录器层次结构中的位置。根logger的名称始终为“”，即空字符串。例如，名为 HTTPServer.RequestHandler 和 HTTPServer.Listener 的logger是记录器 HTTPServer 的后代，而 HTTPServer 本身又是根logger的后代。logger层次结构的深度没有限制。
- 还有一些便捷的宏可用，它们将实际的日志语句包装到检查 Logger 的日志级别是否足以实际记录消息的语句中。
- 例子
```
poco_warning(logger, "This is a warning");
poco_information_f(logger, "An informational message with args: %d, %d", 1, 2);
```
- 析构函数是虚函数
### 成员函数
- static Logger& create(const std::string& name, Channel::Ptr pChannel, int level = Message::PRIO_INFORMATION); 静态成员函数 ：创建并返回一个具有指定名称的 Logger 的引用。Logger 的channel和log level将按指定设置。
- void critical(const std::string & msg); 内联函数：如果 Logger 的日志级别至少为 PRIO_CRITICAL，则创建优先级为 PRIO_CRITICAL 的消息和给定的消息文本，并将其发送到附加通道
- bool critical() const;Returns true if the log level is at least PRIO_CRITICAL.
- void debug(const std::string & msg); 内联函数：如果 Logger 的日志级别至少为 PRIO_DEBUG，则创建一条优先级为 PRIO_DEBUG 的消息，并包含指定的消息文本，并将其发送到连接的通道。
- bool debug() const;Returns true if the log level is at least PRIO_DEBUG.
- static void destroy(const std::string & name);销毁指定名称的记录器。如果未找到该记录器，则不执行任何操作。记录器被销毁后，所有对该记录器的引用都将失效。
- void dump(const std::string & msg,const void * buffer,std::size_t length,Message::Priority prio = Message::PRIO_DEBUG);记录给定的消息，后跟缓冲区中的数据。
- void error(const std::string & msg);内联函数：如果 Logger 的日志级别至少为 PRIO_ERROR，则创建优先级为 PRIO_ERROR 的消息和给定的消息文本并将其发送到附加通道。
- bool error() const;Returns true if the log level is at least PRIO_ERROR.
- void fatal(const std::string & msg);内联函数：如果 Logger 的日志级别至少为 PRIO_FATAL，则创建优先级为 PRIO_FATAL 的消息和给定的消息文本并将其发送到附加通道。
- bool fatal() const;Returns true if the log level is at least PRIO_FATAL.
- static std::string format(const std::string & fmt,const std::string & arg);将 fmt 中所有出现的 $0 替换为 arg 中指定的字符串，并返回结果。要在结果字符串中包含美元符号，请在格式字符串中指定两个美元符号 ($$)。
- static void formatDump(std::string & message,const void * buffer,std::size_t length);对给定的缓冲区创建十六进制转储，并将其附加到给定的消息字符串中
- static Logger & get(const std::string & name);返回指定名称的 Logger 的引用。如果该 Logger 不存在，则基于其父 Logger 创建
- Channel::Ptr getChannel() const;返回连接到logger的通道
- int getLevel() const;内联函数：Returns the Logger's log level.
- static Ptr has(const std::string & name);如果存在，则返回指向具有给定名称的 Logger 的指针，否则返回空指针。
- void information(const std::string & msg);内联函数：如果 Logger 的日志级别至少为 PRIO_INFORMATION，则创建优先级为 PRIO_INFORMATION 的消息和给定的消息文本，并将其发送到附加通道
- bool information() const;Returns true if the log level is at least PRIO_INFORMATION.
- bool is(int level) const;内联函数：如果至少设置了给定的日志级别，则返回 true。
- void log(const Message & msg);虚函数且内联函数：如果给定消息的优先级大于或等于 Logger 的日志级别，则记录该消息。
